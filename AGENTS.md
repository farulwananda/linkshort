## TanStack Intent

Run these from the project root before TanStack-specific changes:

```bash
npx @tanstack/intent@latest list
npx @tanstack/intent@latest load <package>#<skill>
```

Commands run during setup:

```bash
npx @tanstack/intent@latest install
npx @tanstack/intent@latest list
npx @tanstack/intent@latest load @tanstack/start-client-core#start-core
npx @tanstack/intent@latest load @tanstack/react-start#react-start
npx @tanstack/intent@latest load @tanstack/router-plugin#router-plugin
```

Loaded guidance to keep in mind:

- TanStack Start code is isomorphic by default; use `createServerFn` for server-only code.
- Do not use Next.js or Remix patterns.
- Keep `tanstackStart()` before `viteReact()` in `vite.config.ts`.
- Keep `<HeadContent />` in the document head and `<Scripts />` in the document body.
- Let TanStack Router generate `src/routeTree.gen.ts`; do not hand-edit that file.

## Structure

- `src/routes/__root.tsx` owns the document shell.
- `src/routes/index.tsx` is the only app route and intentionally renders an empty `<main />`.
- `src/styles.css` contains only Tailwind imports and base document styles.
- `src/router.tsx` creates the TanStack Router instance.
- `src/routeTree.gen.ts` is generated by `npm run generate-routes`.
